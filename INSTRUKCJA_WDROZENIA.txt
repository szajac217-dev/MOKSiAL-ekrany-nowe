ODŚWIEŻONY EKRAN INFORMACYJNY MOKSiAL / MIASTO / HYBRYDA

1. Co wrzucić do repozytorium

Wgraj do głównego katalogu repozytorium pliki z tej paczki:
- index.html
- config-moksial.json
- config-miasto.json
- config-hybryda.json
- logo-moksial.svg
- qr-moksial.png
- qr-miasto.png
- plakaty-moksial.json
- plakaty-miasto.json
- plakaty-hybryda.json
- generate-posters.yml

Jeżeli w repozytorium workflow GitHub Actions znajduje się w folderze .github/workflows,
plik generate-posters.yml powinien trafić właśnie tam:
.github/workflows/generate-posters.yml

2. Foldery z plakatami

Zostają te same foldery:
- Plakaty-MOKSiAL
- Plakaty-Miasto

Pliki JSON wskazują ścieżki do tych folderów, więc nazwy folderów muszą się zgadzać.

3. Linki do ekranów

Ekran MOKSiAL:
index.html?ekran=moksial

Ekran miasta:
index.html?ekran=miasto

Ekran hybrydowy:
index.html?ekran=hybryda

Dla zgodności zostawiono też obsługę starego parametru:
index.html?ekra=miasto

4. Kody QR

Nowa wersja zawsze pokazuje jednocześnie dwa kody QR:
- qr-moksial.png — wydarzenia MOKSiAL
- qr-miasto.png — wydarzenia miasta

Jeżeli trzeba podmienić kod QR, wystarczy zastąpić odpowiedni plik PNG tą samą nazwą.

5. Czas wyświetlania plakatów

Standardowy plakat wyświetla się 8 sekund.
Plakat z dopiskiem _dluzej w nazwie wyświetla się 30 sekund.

Przykład:
2026-05-20_Kleks_koncert_dzieci.jpg
2026-05-20_Kleks_koncert_dzieci_dluzej.jpg

6. Wagi plakatów

Workflow dalej obsługuje wagi W1-W10.
Im wyższa waga, tym wcześniej plakat pojawi się w kolejce.

Przykład:
2026-05-03_swieto-konstytucji_W10.jpg
2026-05-01_majowka_W5.jpg
2026-05-02_koncert.jpg

Kolejność:
1. najwyższa waga,
2. najbliższa data,
3. nazwa pliku.

7. Pogoda

Pogoda pobierana jest z Open-Meteo dla:
- Szklarskiej Poręby
- Śnieżki

Jeżeli ekran nie ma internetu albo API chwilowo nie odpowiada, kafel pogody automatycznie się ukryje.

8. Ważne uwagi

- Plakaty najlepiej przygotowywać pionowo, np. 1080x1920 px.
- Nazwy plików najlepiej bez polskich znaków, spacji i nietypowych symboli.
- Wersja działa bez dodatkowych bibliotek — tylko HTML, CSS i JavaScript.
- Pliki są przygotowane pod GitHub Pages.
